ProjectMK 리팩토링 마감 전 체크리스트

리팩토링을 끝내기 전에 아래 항목을 다시 확인한다.

1. 호환용 코드 확인

- 호환용 코드는 정말 필요한 경우에만 남긴다.
- 남겨야 하면 왜 남아있는지, 어떤 자산이나 데이터가 의존하는지, 언제 제거 가능한지 주석으로 적는다.
- 그냥 불안해서 남겨두는 코드는 지양한다.

2. 책임 분리 확인

- Manager, Subsystem, Actor, Component, Widget 책임이 다시 섞이지 않았는지 확인한다.
- DataTable, DataAsset, 설정값 접근이 다시 직접 참조로 돌아가지 않았는지 확인한다.
- 위젯 공통 접근은 UMKUserWidget 기준으로 유지되는지 확인한다.

3. 런타임 비용 확인

- Tick 이 불필요하게 늘어나지 않았는지 확인한다.
- delegate binding, unbinding 이 생명주기에 맞게 정리되어 있는지 확인한다.
- UI 갱신은 가능하면 이벤트 기반인지 확인한다.

4. 노출 범위 확인

- EditAnywhere, BlueprintReadWrite 가 필요 이상으로 넓게 열리지 않았는지 확인한다.
- 게임 전체 규칙값은 GameSettingDataAsset 으로 가야 하는지 다시 판단한다.
- 단순 UI 설정값은 BP 노출만으로 충분한지 확인한다.

5. 마감 확인

- 가능하면 빌드나 최소 동작 확인까지 하고 마감한다.
- 후속 작업이 있다면 막연한 TODO 보다는 제거 조건이 보이는 메모로 남긴다.
